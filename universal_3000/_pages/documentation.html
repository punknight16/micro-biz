1. Getting Started

This is a simple platform for managing a web businesses

Start using the application by heading to the login page at "<root>/login". For testing purposes the current version of the application only has 3 users: a root user, an admin user, and a normal user. You can login with any user using credentials provided on the index page. Logging in should provide you with a token_id, a public_key, and a cred_id. These tokens can be used to access all of the functionality provided on the index page, located at "/".

Once logged in, you can create a venture. A venture defines the set of parameters that will prove that your web business has found product-market fit. The product is characterized by the amount of content engagement during a particular time frame. Content engagement is an action a user takes on a given piece of content or web page. It sends a signal to the platform that the content was relevant to your audience in some way. The market is characterized by audience answers to questions in a particular demographic survey. Product-market fit is achieved by showing that a segment of users that took the demographic survey had engagements with the product during a given time, which resulted in sales.

2. Release Info

Version 1.2

Simple html forms were created for the following routes:

  • registration
  • login
  • permissions
  • sales
  • engagements
  • demographics
  • venture
  • analytics

You can access the core objects through the "/list/form" route. (Ex. <root>/permissions/list/form). You can add a core object to the existing core objects throught "/add/form" route. (Ex. <root>/permissions/add/form).

Version 1.3

Logging in causes JWT tokens to be saved in cookies, so you no longer have to type in the token_id, public_key, and cred_id by hand after logging in. You can now access and perform actions on any core object through the "/manager route". (Ex. <root>/permissions/manager).
